# 飲食店予約アプリ（Reservation App）

本アプリは、**飲食店の予約をWeb上で行うことができる予約アプリケーション**です。  
来店客がスマートフォンやPCから予約を行い、店舗側が管理画面で予約を確認・管理することを想定して開発しました。

Webアプリ開発における  
**「画面設計 → バックエンド処理 → DB設計 → AWSデプロイ」**  
までを一通り経験することを目的として制作しました。

---

## 公開URL（実際に操作できます）

### 予約ページ（一般ユーザー向け）
http://reservation-app.duckdns.org

来店客が使用することを想定した予約ページです。

- 予約日時・人数を指定
- お名前・連絡先を入力
- 確認画面 → 予約完了画面へ遷移

という **実際の飲食店予約フローを再現** しています。

---

### 管理画面（店舗スタッフ向け）
http://reservation-app.duckdns.org/reserve/list

店舗スタッフが予約状況を確認するための画面です。  
登録された予約を一覧で確認し、編集・削除が可能です。

※ 本ポートフォリオでは、  
ユーザー向け画面と管理画面をURL単位で分離し、  
役割ごとに画面を分けたシンプルな構成としています。  
将来的に認証機能などを追加しやすい設計を意識しました。

## 主な機能

### 予約機能
- 日付時間選択
- 人数指定
- 予約情報入力
- 入力内容確認画面
- 予約完了画面表示

### 管理者機能
- 予約一覧表示
- 予約編集
- 予約削除

---

## カレンダー機能について（工夫した点）

インフォメーションページでは、予約状況をカレンダー形式で表示し、  
ユーザーが空き日を視覚的に判断しやすいUIを意識しました。  
※ 現在は表示機能を中心とし、今後フォーム連携などの拡張が可能な構成としています。

- 日付クリックで予約ページへ遷移（空き日の確認を目的）
- 管理画面の予約データと連動

バックエンドでは、  
**日付単位で予約情報を管理する設計** にしており、  
将来的に「満席日表示」や「予約制限」機能を追加できる構成です。

## データベース設計（reservations）

| カラム名 | 内容 |
|--------|------|
| id | 予約ID |
| name_kanji | 氏名（漢字） |
| name_kana | 氏名（カナ） |
| phone_number | 電話番号 |
| email | メールアドレス |
| date | 予約日 |
| time | 予約時間 |
| number_of_people | 人数 |
| created_at | 登録日時 |

---

## デプロイ構成

- Spring Boot アプリを JAR 化
- S3 経由で EC2 に配置
- systemd により常時起動

本番環境を意識し、  
**再起動後もアプリが自動起動する構成** にしています。
